SIMPLIFIED XML SPECIFICATION (the format you generate)
vs. full Cisco Packet Tracer 6.2 XML structure (the program builds it).

===============================================================================
A. SIMPLIFIED XML — WHAT YOU GENERATE
===============================================================================

Root element: <network>
  Children:
    <devices>  — one or more <device> elements
    <links>    — zero or more <link> elements

-------------------------------------------------------------------------------
A.1 <device>
-------------------------------------------------------------------------------
Attributes:
  name    (required, string, unique within the network)
  type    (required): router | switch | pc | laptop | server | printer |
                      ip_phone | hub | bridge | repeater | access_point |
                      wireless_router | cloud | modem | voip | tablet |
                      smartphone | tv | firewall | sniffer
  model   (recommended): exact model string from the device reference.
                         If omitted, defaults are used (router→1841,
                         switch→2960-24TT, server→Server-PT, hub→Hub-PT, …).
  x, y    (optional): logical-canvas coordinates. Default: 200×150 grid
                      starting at (100, 100).
  gateway (optional): default gateway for end-hosts.

Children:
  <interface> ... </interface>          — 0..N
  <vlans> <vlan .../> ... </vlans>      — 0..1 (switches only)
  <config> <line>…</line> ... </config> — 0..1 running-config (router/switch)
  <services> ... </services>            — 0..1 (servers only)
  <modules> ... </modules>              — 0..1 (declarative; not yet applied)

-------------------------------------------------------------------------------
A.2 <interface>
-------------------------------------------------------------------------------
Attributes:
  name       (required): IOS-style port name.
                          FastEthernet0/0, GigabitEthernet0/0, Serial0/0/0,
                          Ethernet0/1/0, FastEthernet0 (PC), Wireless0,
                          "Port 0" (hub).
  ip         (string, IPv4)
  subnet     (string, e.g. 255.255.255.0)
  gateway    (IPv4, end-hosts only)
  clockrate  (bps; Serial DCE only — usually 2000000 or 64000)
  bandwidth  (kbps)
  mac        (XXXX.XXXX.XXXX)
  duplex     (full | half | auto)
  vlan       (1..4094) — VLAN number for switch access port
  trunk      ("true") — switch trunk port

-------------------------------------------------------------------------------
A.3 <vlans> / <vlan>
-------------------------------------------------------------------------------
Switch only. Default system VLANs (1, 1002–1005) are added automatically.
  <vlan number="10" name="staff"/>

-------------------------------------------------------------------------------
A.4 <config> / <line>
-------------------------------------------------------------------------------
Multi-line running-config. One IOS command per <line>. The program copies
this verbatim into <RUNNINGCONFIG>/<LINE> in the device's ENGINE block.

-------------------------------------------------------------------------------
A.5 <services>  (server only)
-------------------------------------------------------------------------------
Boolean services (single attribute `enabled`):
  <http enabled="true|false"/>
  <https enabled="true|false"/>
  <ftp enabled="true|false"/>
  <tftp enabled="true|false"/>
  <ntp enabled="true|false"/>
  <syslog enabled="true|false"/>
  <email enabled="true|false"/>   (controls SMTP_ENABLED + POP3_ENABLED)

DHCP server:
  <dhcp enabled="true">
    <pool name="LAN" network="10.0.0.0" mask="255.255.255.0"
          default_router="10.0.0.1" dns="10.0.0.50"
          start="10.0.0.100" end="10.0.0.200"
          max_users="512" domain=""/>
  </dhcp>

DNS server:
  <dns enabled="true">
    <record name="host.example.com" type="A" ip="10.0.0.50"/>
    <record name="alias" type="CNAME" hostname="host.example.com"/>
  </dns>

The simplified DNS attribute `type="A"` is converted to PT's "A-REC". Each
record is rendered as a <RESOURCE-RECORD> with TYPE/NAME/TTL and either
IPADDRESS (for A-REC) or HOSTNAME (for CNAME / NS).

-------------------------------------------------------------------------------
A.6 <link>
-------------------------------------------------------------------------------
Attributes:
  from, to            (required) — device names from <device name="...">
  from_port, to_port  (required) — interface names on each end
  type                (optional) — copper (default) | crossover | serial |
                                   fiber | coaxial | phone
                       crossover — copper cross-over (router↔router etc.)

===============================================================================
B. FULL CISCO PT XML STRUCTURE  (FYI — the program builds it for you)
===============================================================================

Root:  <PACKETTRACER5> ... </PACKETTRACER5>
Required version: 6.2.0.0052.

Sections:
  <VERSION>              constant "6.2.0.0052"
  <PIXMAPBANK/>          empty
  <NETWORK>              ← your devices and links go here
    <DEVICES>            list of <DEVICE>
    <LINKS>              list of <LINK>
    <SHAPETESTS/>
    <DESCRIPTION translate="true"/>
  <SCENARIOSET>          "Scenario 0"
  <OPTIONS>              UI / language / certificates / script modules
  <PHYSICALWORKSPACE>    Intercity / Home City / Corporate Office / Closet
  <FILTERS>              ICMP, OSPF, BGP, etc.
  <CLUSTERS>             RootCluster
  <LINES/>, <RECTANGLES/>, <ELLIPSES/>, <POLYGONS/>
  <USER_PROFILE>         Guest
  <MULTIUSER>
  <GEOVIEW_GRAPHICSITEMS/>
  <SCRIPT_MODULE>
  <CEPS/>

-------------------------------------------------------------------------------
<DEVICE>
  <ENGINE>
    <TYPE customModel="" model="<MODEL>">Router|Switch|Pc|Server|Hub|...</TYPE>
    <NAME translate="true">hostname</NAME>
    <POWER>true</POWER>
    <DESCRIPTION/>
    <MODULE>
      <TYPE>...</TYPE>
      <SLOT>
        <TYPE>...</TYPE>
        <MODULE>
          <MODEL>...</MODEL>
          <PORT>
            <TYPE>eCopperFastEthernet|eGigabitEthernet|eSerial|...</TYPE>
            <MACADDRESS>...</MACADDRESS>
            <BIA>...</BIA>
            <IP>10.0.0.1</IP>           ← per-port IP
            <SUBNET>255.255.255.0</SUBNET>
            <PORT_GATEWAY/>
            <IPV6_LINK_LOCAL>FE80::...</IPV6_LINK_LOCAL>
            ... (TIMEOUT, FIREWALL flags, etc.)
          </PORT>
          ... more ports ...
        </MODULE>
      </SLOT>
    </MODULE>

    <!-- routers / switches -->
    <RUNNINGCONFIG>  <LINE>...</LINE>  ... </RUNNINGCONFIG>
    <STARTUPCONFIG>  ... </STARTUPCONFIG>
    <FILE_MANAGER> ... flash:/IOS image ... </FILE_MANAGER>

    <!-- switches -->
    <VLANS> <VLAN number="1" name="default"/> ... </VLANS>
    <VTP> <DOMAIN_NAME/> <MODE>0</MODE> ... </VTP>
    <SECURITY> ... </SECURITY>

    <!-- end-hosts -->
    <GATEWAY>10.0.0.1</GATEWAY>
    <DNS_CLIENT> <SERVER_IP/> </DNS_CLIENT>
    <EMAIL_CLIENT> ... </EMAIL_CLIENT>
    <WIRELESS_CLIENT> ... </WIRELESS_CLIENT>

    <!-- servers (Server-PT) -->
    <HTTP_SERVER> <ENABLED>1</ENABLED> </HTTP_SERVER>
    <HTTPS_SERVER> <HTTPSENABLED>1</HTTPSENABLED> </HTTPS_SERVER>
    <DNS_SERVER>
      <ENABLED>1</ENABLED>
      <NAMESERVER-DATABASE>
        <RESOURCE-RECORD>
          <TYPE>A-REC</TYPE>
          <NAME>example.com</NAME>
          <TTL>86400</TTL>
          <IPADDRESS>10.0.0.50</IPADDRESS>
        </RESOURCE-RECORD>
      </NAMESERVER-DATABASE>
    </DNS_SERVER>
    <DHCP_SERVERS>
      <ASSOCIATED_PORTS>
        <ASSOCIATED_PORT>
          <NAME>FastEthernet0</NAME>
          <DHCP_SERVER>
            <ENABLED>1</ENABLED>
            <POOLS> <POOL> ... </POOL> </POOLS>
          </DHCP_SERVER>
        </ASSOCIATED_PORT>
      </ASSOCIATED_PORTS>
    </DHCP_SERVERS>
    <FTP_SERVER> <ENABLED>1</ENABLED> ... </FTP_SERVER>
    ... and so on.

    <SYS_NAME>...</SYS_NAME>
    <SERIALNUMBER>...</SERIALNUMBER>
  </ENGINE>

  <WORKSPACE>
    <LOGICAL>
      <X>200.0</X>
      <Y>200.0</Y>
      <DEVCLUSTERID>1-1</DEVCLUSTERID>
    </LOGICAL>
    <PHYSICAL translate="true">Intercity,Home City,...,Rack,<name></PHYSICAL>
  </WORKSPACE>
</DEVICE>

-------------------------------------------------------------------------------
B.2 <LINK>  — full structure
-------------------------------------------------------------------------------
<LINK>
  <TYPE>eCopper | eFiber | eSerial | eCoaxial | ePhone</TYPE>
  <CABLE>
    <LENGTH>1.0</LENGTH>
    <FROM>0</FROM>                    <!-- 0-based device index in <DEVICES> -->
    <PORT>FastEthernet0/0</PORT>      <!-- port name on FROM side -->
    <TO>1</TO>                        <!-- 0-based device index -->
    <PORT>FastEthernet0/1</PORT>      <!-- port name on TO side -->
    <GEO_VIEW_COLOR>#000000</GEO_VIEW_COLOR>
    <TYPE>eStraightThrough | eCrossOver</TYPE>
  </CABLE>
</LINK>

The program looks up FROM/TO indices automatically by matching <link from=>/
<link to=> to <device name=>, so you never write numeric indices.

===============================================================================
C. REQUIRED VS OPTIONAL FIELDS
===============================================================================

REQUIRED in your simplified XML:
  <device> needs name + type. Without model the program picks a default.
  <link>   needs from + from_port + to + to_port.

OPTIONAL — the program fills in sensible values:
  device   x/y, gateway, model
  port     mac (auto-generated), bandwidth (template default), clockrate
  link     type ("copper" by default)

The program ALWAYS provides MEM_ADDR / DEV_ADDR / SERIALNUMBER / BIA /
IPV6_LINK_LOCAL / FILE_MANAGER / SECURITY / VTP / etc. — never write these
yourself.
